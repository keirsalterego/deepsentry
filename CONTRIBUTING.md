# Contributing to DeepSentry

Thanks for helping improve DeepSentry.

## What to contribute

- Bug fixes
- Documentation improvements
- New tests
- Safer config handling
- Smaller, clearer file and module names

## Before you submit

- Keep changes focused.
- Preserve the MIT license and third-party attribution.
- Avoid introducing new runtime assumptions unless they are documented.
- Update the README or docs when behavior changes.

## Suggested checks

- Run the relevant Docker or Python entry point.
- Verify imports still resolve after renames.
- Confirm the book-style docs still open in a browser.
