Troubleshooting

The most common issues are missing data paths, missing dependencies, or a mismatch between the expected and actual YAML file names.

Checklist

  • Confirm the data root contains dockerconfig
  • Confirm the renamed config files still exist
  • Confirm the Docker paths match the new short folder names
  • Confirm pytest is installed in the local venv when running tests

If the project fails during encoding, inspect the input dataset for empty or malformed log lines. If live monitoring stalls, verify the configured data file still has readable log entries and the time interval is appropriate.
